Production NLP Categorization Microservice프로덕션 NLP 카테고리 매칭 마이크로서비스

A live AI module matching natural-language user inputs to internal DB categories.자연어 사용자 입력을 내부 DB 카테고리에 매칭하는 라이브 AI 모듈.

Decision

Implemented an embedding + reranking pipeline (BGE-m3) rather than keyword matching, to maximize accuracy of natural-language matching against static categorical data.키워드 매칭이 아니라 embedding + reranking 파이프라인(BGE-m3)으로 구현해, 정적 카테고리 데이터 대비 자연어 매칭 정확도를 극대화했습니다.

Why

To make the core backend team able to adopt it with zero restructuring, I shipped it containerized (Docker) with full Swagger docs — a plug-and-play service, not a code dependency.코어 백엔드 팀이 재구조화 없이 도입할 수 있도록, Docker 컨테이너 + Swagger 문서로 전달했습니다 — 코드 의존성이 아니라 plug-and-play 서비스로.

Impact

Plugged into the live commercial environment without backend restructuring.백엔드 재구조화 없이 상용 환경에 그대로 투입.

Large-Scale Asset Generation & Review Pipeline대규모 에셋 생성·검수 파이프라인

Mass-generation and review of 22,000+ service image assets, under strict hardware limits.엄격한 하드웨어 제약 아래 22,000+ 서비스 이미지 에셋 대량 생성·검수.

Decision

For a one-off task I intentionally excluded heavy DBs / message queues to avoid over-engineering, and used the physical hardware I/O limit as a natural rate limiter for a stable synchronous flow.일회성 작업이라 over-engineering을 피하려고 무거운 DB·MQ를 의도적으로 배제하고, 물리 하드웨어 I/O 한계를 자연스러운 rate limiter로 활용해 안정적인 동기 흐름을 유지했습니다.

Why

The review dashboard rendered 22,000+ entries in over 5 minutes (CSR bottleneck); I cut it to near-instant with lazy loading and browser caching instead of a heavier rebuild.검수 대시보드가 22,000+ 항목을 5분 넘게 렌더(CSR 병목)했는데, 무거운 재구축 대신 lazy loading·브라우저 캐싱으로 거의 즉시로 단축했습니다.

Scaling

Ran PoC on local Flux, then migrated mass production to the Gemini API for stability and speed.로컬 Flux로 PoC 후, 안정성·속도를 위해 대량 생산을 Gemini API로 이전했습니다.

Zero-Overhead Internal Automation Pipeline제로 오버헤드 내부 자동화 파이프라인

An internal-only automation hub collecting scattered static data and dispatching notifications.흩어진 정적 데이터를 모아 알림을 발송하는 내부 전용 자동화 허브.

Decision

Security policy blocked external cloud-DB access. Instead of requesting new APIs from the core team, I reverse-engineered the internal CMS and reused legacy APIs (email/SMS dispatch) — zero additional backend resources.보안 정책으로 외부 클라우드 DB 접근이 막혔습니다. 코어 팀에 새 API를 요청하는 대신 내부 CMS를 리버스 엔지니어링해 레거시 API(이메일/SMS 발송)를 재사용했습니다 — 추가 백엔드 자원 0.

Why

To stay compliant, scripts run strictly inside the closed internal network (no external port-forwarding), inheriting the existing modules' security authentication.보안 준수를 위해 스크립트를 폐쇄 내부망 안에서만 실행(외부 포트포워딩 없음)하고, 기존 모듈의 보안 인증을 그대로 상속했습니다.

Lightweight Internal Network Utility경량 내부망 유틸리티

A small script to resolve a corporate VPN bottleneck.사내 VPN 병목을 해소하는 작은 스크립트.

Decision

Security policy restricted simultaneous VPN connections, bottlenecking the team. I built and deployed a lightweight Python port-connection script enabling multiple secure connections with minimal setup.보안 정책이 동시 VPN 연결을 제한해 팀 작업이 병목됐습니다. 경량 Python 포트 연결 스크립트를 만들어 최소 설정으로 다중 보안 연결을 가능하게 배포했습니다.

Impact

Drastically improved team productivity with minimal setup.최소 설정으로 팀 생산성을 크게 개선.
